The Elements

a

The HTML <a> element (or anchor element) creates a hyperlink to other web pages, files, locations within the same page, email addresses, or any other URL. MDN web docs: <a>: The Anchor element

Example

Output
Link to Named Anchor
Named Anchor
Code
<!-- anchor linking to external file -->
<a href="#anchor-example" title="Go to the Named Anchor">Link to Named Anchor</a>
<br>
<!-- placeholder links/named anchors (without href) --><a name="anchor-example">Named Anchor</a>

abbr

The HTML Abbreviation element(<abbr>) represents an abbreviation or acronym; the optional title attribute can provide an expansion or description for the abbreviation. If present, title must contain this full description and nothing else. MDN web docs: <abbr>: The Abbreviation element

Example

Output
Using HTML is fun and easy!
Code
Using <abbr title="HyperText Markup Language">HTML</abbr> is fun and easy!

address

The HTML <address> element indicates that the enclosed HTML provides contact information for a person or people, or for an organization. MDN web docs: <address>: The Contact Address element

Example

Output

Code
<address>Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</address>

article

The HTML <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). Examples include: a forum post, a magazine or newspaper article, or a blog entry. MDN web docs: <article>: The Contact Address element

Example

Output
Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
Code
<article>Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</article>

aside

The HTML <aside> element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes. MDN web docs: <aside>: The Aside element

Example

Output

b

The HTML Bring Attention To element (<b>)  is used to draw the reader's attention to the element's contents, which are not otherwise granted special importance. This was formerly known as the Boldface element, and most browsers still draw the text in boldface. However, you should not use <b> for styling text; instead, you should use the CSS font-weight property to create boldface text, or the <strong> element to indicate that text is of special importance. MDN web docs: <b>: The Bring Attention To element

Example

Output
This article describes several text-level elements.
Code
This article describes several <b>text-level</b> elements.

bdi

The HTML Bidirectional Isolate element (<bdi>)  tells the browser's bidirectional algorithm to treat the text it contains in isolation from its surrounding text. It's particularly useful when a website dynamically inserts some text and doesn't know the directionality of the text being inserted.

MDN web docs: <bdi>: The Bidirectional Isolate element

Example

Output
اَلأَعْشَى - 1st place
Jerry Cruncher - 2nd place
Code
<bdi>اَلأَعْشَى</bdi> - 1st place
<br>
<bdi>Jerry Cruncher</bdi> - 2nd place

bdo

The HTML Bidirectional Text Override element (<bdo>) overrides the current directionality of text, so that the text within is rendered in a different direction.MDN web docs: <bdo>: The Bidirectional Text Override element

Example

Output

This text will go left to right.

This text will go right to left.

Code
<!-- Switch text direction -->
<p>This text will go left to right.</p>
<p><bdo dir="rtl">This text will go right to left.</bdo></p>

blockquote

The HTML <blockquote> Element (or HTML Block Quotation Element) indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation. A URL for the source of the quotation may be given using the cite attribute, while a text representation of the source can be given using the <cite> element. MDN web docs: <blockquote>: The Block Quotation element

Example

Output

Avian carriers can provide high delay, low throughput, and low altitude service. The connection topology is limited to a single point-to-point path for each carrier, used with standard carriers, but many carriers can be used without significant interference with each other, outside of early spring. This is because of the 3D ether space available to the carriers, in contrast to the 1D ether used by IEEE802.3. The carriers have an intrinsic collision avoidance system, which increases availability.

Code
<blockquote cite="https://tools.ietf.org/html/rfc1149"><p>Avian carriers can provide high delay, low throughput, and low altitude service. The connection topology is limited to a single point-to-point path for each carrier, used with standard carriers, but many carriers can be used without  significant interference with each other, outside of early spring. This is because of the 3D ether space available to the carriers, in contrast to the 1D  ether used by IEEE802.3. The carriers have an intrinsic collision avoidance system, which increases availability.</p></blockquote>

cite

The HTML Citation element (<cite>) is used to describe a reference to a cited creative work, and must include the title of that work. The reference may be in an abbreviated form according to context-appropriate conventions related to citation metadata. MDN web docs: <cite>: The Citation element

Example

Output
More information can be found in [ISO-0000].
Code
More information can be found in <cite>[ISO-0000]</cite>.

code

The HTML <code> element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code. By default, the content text is displayed using the user agent's default monospace font. MDN web docs: <code>: The Inline Code element

Example

Output
The function selectAll() highlights all the text in the input field so the user can, for example, copy or delete the text.
Code
The function <code>selectAll()</code> highlights all the text in the input field so the user can, for example, copy or delete the text.

data

The HTML <data> element links a given content with a machine-readable translation. If the content is time- or date-related, the <time> element must be used. MDN web docs: <data>: The Machine-Readable data element

Example

Output

New Products

  • Mini Ketchup
  • Jumbo Ketchup
  • Mega Jumbo Ketchup
Code
<p>New Products</p>
<ul>
	<li><data value="398">Mini Ketchup</data></li>
	<li><data value="399">Jumbo Ketchup</data></li>
	<li><data  value="400">Mega Jumbo Ketchup</data></li>
</ul>

del

The HTML <del> element represents a range of text that has been deleted from a document. This can be used when rendering "track changes" or source code diff information, for example. The <ins> element can be used for the opposite purpose: to indicate text that has been added to the document. MDN web docs: <del>: The Deleted Text data element

Example

Output

This text has been deleted, here is the rest of the paragraph.

Code
<p><del>This text has been deleted</del>, here is the rest of the paragraph.</p>

dfn

The HTML Definition element (<dfn>) is used to indicate the term being defined within the context of a definition phrase or sentence. The <p> element, the <dt>/<dd> pairing, or the <section> element which is the nearest ancestor of the <dfn> is considered to be the definition of the term. MDN web docs: <dfn>: The Definition element

Example

Output
The HTML Definition element (<dfn>) is used to indicate the term being defined within the context of a definition phrase or sentence.

The HST is among the most productive scientific instruments ever constructed.
Code
<!-- Basic identification of a term -->
The <strong>HTML Definition element</strong> (<strong><dfn>&lt;dfn&gt;</dfn></strong>) is used to indicate the term being defined within the context of a definition phrase or sentence.
<br><br>
<!-- Using abbreviations and definitions together -->
The <dfn><abbr title="Hubble Space Telescope">HST</abbr></dfn> is among the most productive scientific instruments ever constructed.

div

The HTML Content Division element (<div>) is the generic container for flow content. It has no effect on the content or layout until styled using CSS. MDN web docs: <div>: The Content Division element

dl

The HTML <dl> element represents a description list. The element encloses a list of groups of terms (specified using the <dt> element) and descriptions (provided by <dd> elements). Common uses for this element are to implement a glossary or to display metadata (a list of key-value pairs). MDN web docs: <dl>: The Description List element

Gigantis
Godzilla

Japan
Color
Green

	<dd>Gigantis</dd>
	<dd>Godzilla</dd>
	<dt>Born</dt>
	<dd>1952</dd>
	<dt>Birthplace</dt>
	<dd>Japan</dd>
	<dt>Color</dt>
	<dd>Green</dd>
</dl>

em

The HTML <em> element marks text that has stress emphasis. The <em> element can be nested, with each level of nesting indicating a greater degree of emphasis. MDN web docs: <em>: The Emphasis element

Example

Output
In HTML 5, what was previously called block-level content is now called flow content.
Code
In HTML 5, what was previously called <em>block-level</em> content is now called <em>flow</em> content.

i

The HTML <i> element represents a range of text that is set off from the normal text for some reason. Some examples include technical terms, foreign language phrases, or fictional character thoughts. It is typically displayed in italic type. MDN web docs: <i>: The Set-off Text element

Example

Output
The Latin phrase Veni, vidi, vici is often mentioned in music, art, and literature.
Code

ins

The HTML <ins> element represents a range of text that has been added to a document. You can use the <del> element to similarly represent a range of text that has been deleted from the document. MDN web docs: <ins>: The Inserted Text element

Example

Output
This text has been inserted
Code
<ins>This text has been inserted</ins>

kbd

The HTML Keyboard Input element (<kbd>) represents a span of inline text denoting textual user input from a keyboard, voice input, or any other text entry device. By convention, the user agent defaults to rendering the contents of a <kbd> element using its default monospace font, although this is not mandated by the HTML standard. MDN web docs: <kbd>: The Keyboard Input element

Example

Output
Use the command help mycommand to view documentation for the command "mycommand".
Code
Use the command <kbd>help mycommand</kbd> to view documentation for the command "mycommand".

main

The HTML <main> element represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application. MDN web docs: <main>: The Main sectioning element

Output

pre

The HTML <pre> element represents preformatted text which is to be presented exactly as written in the HTML file. The text is typically rendered using a non-proportional ("monospace") font. Whitespace inside this element is displayed as written. MDN web docs: <pre>: The Paragraph element

}

q

The HTML <q> element indicates that the enclosed text is a short inline quotation. Most modern browsers implement this by surrounding the text in quotation marks. This element is intended for short quotations that don't require paragraph breaks; for long quotations use the <blockquote> element. MDN web docs: <q>: The Inline Quotation element

Example

Output
According to Mozilla's website, Firefox 1.0 was released in 2004 and became a big success.
Code
According to Mozilla's website, <q cite="https://www.mozilla.org/en-US/about/history/details/">Firefox 1.0 was released in 2004 and became a big success.</q>

s

The HTML <s> element renders text with a strikethrough, or a line through it. Use the <s> element to represent things that are no longer relevant or no longer accurate. However, <s> is not appropriate when indicating document edits; for that, use the <del> and <ins> elements, as appropriate. MDN web docs: <s>: The Strikethrough element

Example

Output
Today's Special: Salmon SOLD OUT
Code
<s>Today's Special: Salmon</s> SOLD OUT

samp

The HTML Sample Element (<samp>) is used to enclose inline text which represents sample (or quoted) output from a computer program. Its contents are typically rendered using the browser's default monospaced font (such as Courier or Lucida Console). MDN web docs: <samp>: The Sample Output element

small

The HTML <small> element makes the text font size one size smaller (for example, from large to medium, or from small to x-small) down to the browser's minimum font size.  In HTML5, this element is repurposed to represent side-comments and small print, including copyright and legal text, independent of its styled presentation. MDN web docs: <small>: The Small element

Example

Output
This is the first sentence. This whole sentence is in small letters.
Code
This is the first sentence. <small>This whole sentence is in small letters.</small>

span

The HTML <span> element is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang. It should be used only when no other semantic element is appropriate. <span> is very much like a <div> element, but <div> is a block-level element whereas a <span> is an inline element. MDN web docs: <span>: The Small element

Example

Output

wbr

The HTML <wbr> element represents a word break opportunity—a position within text where the browser may optionally break a line, though its line-breaking rules would not otherwise create a break at that location. MDN web docs: <wbr>: The Word Break Opportunity element
